How Does a Computer Remembers?

This concept is really simple, and is composed by a clever analogy:

All of us have light switches at home. Some may be the classical ones, where you need to toggle it with your fingers, and others have Bluetooth and wifi capabilites. I'll leave to you which one to imagine.

The use of a switch is the most classical way to make your house remembers a state. You can turn the kitchen on, the livingroom off, and the bedrooms on. If you leave your house, and come back some time later, you'll find yourself with your house on the same state it was before: With kitchen on, the livingroom off, and the bedrooms on. This is the same way computer memory works: It uses swithces (but on their case eletrical, not mechanical) to remember the state of what was written before into it. They use a switch called transistor, which is a switch you can toggle using eletrical energy. If you want to turn it on, you put energy into it, if you want to turn it off, you remove energy from it[1].

What is RAM and why we are so dependent of it

The computer needs a set of these transistors switches to be able to remember states. This is required because not only it needs to remember where it was in a specific point in time but also to successfully proceed with a set of instructions gave to it. It also needs a way of storing and retrieving data from the user, otherwise, it would forget your name every time.

There's a catch, tho.

Multiple types of these memories are used and they have different objectives and characteristics, so the computer knows when to use which one, and for an specific puprpose. The list is not too big, but worth knowing. The fact that the memory hardware persists data when turned off is refered as volatile. Memory that persists data after turned off is caled non-volatile, and memory that looses the data after loss of power is called volatile. The table below summarizes the main types of memory used in computers, their volatility, speed, typical use-cases, and some notes on their characteristics.

Memory Type Volatility Speed Typical Use‑Case Notes
Registers Volatile Fastest Hold data for immediate processing by the CPU Extremely limited capacity since they are part of the CPU itself
DRAM Volatile Slower than SRAM (requires refresh) Main system memory (RAM) e.g., DDR variants; needs refresh
Flash (NAND/EEPROM) Non‑volatile Moderate (fast read, slow write/erase) SSDs, USB drives, firmware Fast than HDDs but slower than RAM
Magnetic storage (HDD) Non‑volatile Slow (mechanical) Mass storage Large capacity and cost-effective

So what is RAM?

RAM is is volatile memory that requires constant refreshing to maintain its data, and is used as the main system memory for running applications and the operating system. The reason why we're so dependent on it is because it provides a fast and efficient way for the CPU to access data and instructions that are currently in use, from programs that you might be using, and to hold instructions of what the CPU needs to do next.

Without RAM, the computer would have to rely on much slower storage options for the basic calculations it need to do, and to storage the objects needed for specific things while programs are running, which would significantly degrade performance and make it impractical for most tasks. If we decided to add flash and have a big memory instead, our performance would downgrade, and if we decided to have those as registers built inside the CPU, we would have to have way more expansive and bigger CPUs. RAM comes right in the middle: A way to have a few GBs of memory that are fast enough for our computer to work without trading off read/write speed.

How does RAM helps the computer?

The answer is pretty straight foward. It sits there as a safe space for the computer to write data and retreive when it needs to continue. The RAM is managed by the operating system, which allocates (or grabs) the amount of memory necessary for a task during a program execution. Let's say that you load a program that translate text.

During the first initialization of the program, it asks for some memory to work, let's say, 1Kb. It's going to use this 1Kb to store the text you want to translate. After you type the text and press translate, the software then asks the operating system to save this into memory, which is then used to refer as the base text for the translation. After your program translates it (we're not going to dive into the translation process here on this article for obvious reasons), the program asks the OS to then allocate another 1Kb for the results, fetches this data and present to you on the screen.

This example highlights the importance of the RAM in your system. It helps the computer to storage intermediary results so it those can be used when it most needs. After you shut down your computer, and does not save the translated text into a .txt file (in this case, using the HD or the Flash memory), it will be lost, because as we've seen in the begining, RAM is volatile.

How it works under the hood

As we stated earlier, RAM uses a bunch of switches. The curious thing, tho, is that we can access one switch at the time for every single chip of RAM[2].

To be able to fetch this state, and since a RAM chip has tons of these switches, we need to tell the RAM chip in what address the switch we want to access is. After we tell the chip where it is, it will tell us the state of the switch: Positive voltage when ON, 0 volts when OFF. This is normally read in a digital system by 1 when ON and 0 when OFF.

Since RAM chips constantly have grown in size (thus making it possible to access more addresses), it was clear that the amount of address pins needed to be increased. If you wanted to have an address bus (the sequence of address pins) to be able to access 64Kbs of memory, you would need 16 pins. In binary, these 16 pins, in binary, would give you the possibility to access just this amount. But what if we make a RAM chip bigger than 64Kb?

The answer is logic: We need more addressing pins. on the figure below, we can count a total of 20 address pins, which let you address around 1 Billion different combinations, or 1Gb.

It sounds impressive right? In today's world, a 1Gb of RAM cannot do much, but we're talking about being able to store 1 Billion different combinations of switches into it.

But as you can see, this RAM chip can only hold either a 1 or a 0 in it's switch. Once you pass that address to the RAM iteself, you'll be able to fetch a single bit. How can we store a byte (which are a sequence of 8 bits) of information then? We can do it sequentially, or parallely, which seems more clever.

If you connect 8 of those 1Gb of RAM paralelly on the address lines, you'll end up with all the same amount of address pins, but with now 8 data lines. If you correctly build your accessing logic, when you send the address you want to the address pin, now you can store a whole byte into it by adding to the different data lines the bits that you need. You can now save 8 Billion bits, or 1 Billion bytes.

Computers engineers can now work on serving (or retreiving) to the RAM chip the smallest unit possible (the byte). This happens because sotwares do not tend to save only one or two bits. For that, we request a memory that we can save a whole byte, and use the byte as the smallest unit possible. That's why a RAM chip that can store 8 Billion bits (1 Billion bytes), is a 1Gb RAM.
